## 二、三个 Agent 的职责

### 2.1 主 Agent (Co) - 生产环境

| 属性 | 值 |
|------|-----|
| 名称 | Co |
| 端口 | 8088 |
| 工作目录 | /root/.copaw |
| 角色 | 生产环境、与用户交互、验收决策 |

**职责**：
- 与 Derek 交互，理解需求
- 分配任务给 DevAgent
- 接收 TestAgent 的测试报告
- 做验收决策（是否升级生产）
- 管理整体流程

### 2.2 DevAgent - 开发环境

| 属性 | 值 |
|------|-----|
| 名称 | DevAgent |
| 端口 | 8089 |
| 工作目录 | /root/.copaw_dev |
| 容器 | copaw-dev |
| 角色 | 代码开发、仓库维护 |

**职责**：
- 维护 Fork 仓库与官方仓库同步（rebase）
- 保留我们的自定义改动（session 继承等）
- 开发新功能
- 将代码推送到 Fork 仓库
- 通知 TestAgent 执行测试

### 2.3 TestAgent - 测试环境

| 属性 | 值 |
|------|-----|
| 名称 | TestAgent |
| 端口 | 8091 |
| 工作目录 | /root/.copaw_test |
| 容器 | copaw-test |
| 角色 | 测试执行、质量保障 |

**职责**：
- 执行功能测试
- 执行性能测试
- 执行压力测试
- 生成测试报告
- 通知主 Agent 测试结果

## 四、仓库同步策略

### 4.1 分支结构

```
Fork 仓库 (derekjchen/CoPaw)
├── main              # 与官方同步的基础分支
├── derek/main        # Derek 的主分支（包含自定义改动）
├── derek/session     # Session 继承功能
├── derek/memory      # 记忆系统增强
└── feature/xxx       # 新功能分支
```

### 4.2 同步流程

```
1. 检查官方仓库更新 (每小时)
   git fetch origin

2. 检查是否有新提交
   git log HEAD..origin/main --oneline

3. 如果有更新，执行 rebase
   git checkout main
   git rebase origin/main

4. 将自定义改动 rebase 到最新
   git checkout derek/main
   git rebase main

5. 解决冲突（如有）
   - DevAgent 通知 Co
   - Co 请 Derek 决策

6. 推送到 Fork
   git push fork derek/main
```

## 七、安全考虑

1. **密钥管理**
   - Token 存储在 DevAgent 容器内
   - 其他 Agent 无法访问

2. **代码安全**
   - 只有 DevAgent 能 push 代码
   - 只有通过测试的代码才能 push

3. **环境隔离**
   - 每个容器独立网络
   - 生产环境不暴露开发端口
