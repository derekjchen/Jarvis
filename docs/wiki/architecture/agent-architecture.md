## 三个 Agent 的职责

### 主 Agent (Co)

| 属性 | 值 |
|------|-----|
| 名称 | Co |
| 端口 | 8088 |
| 工作目录 | /root/.copaw |
| 角色 | 生产环境、与用户交互、验收决策 |

**职责**：
- 与 Derek 交互，理解需求
- 分配任务给 DevAgent
- 接收 TestAgent 的测试报告
- 做验收决策（是否升级生产）
- 管理整体流程

### DevAgent

| 属性 | 值 |
|------|-----|
| 名称 | DevAgent |
| 端口 | 8089 |
| 工作目录 | /root/.copaw_dev (容器内) |
| 容器 | copaw-dev |
| 角色 | 代码开发、仓库维护 |

**职责**：
- 维护 Fork 仓库与官方仓库同步（rebase）
- 保留自定义改动（session 继承等）
- 开发新功能
- 推送代码到 Fork 仓库
- 通知 TestAgent 执行测试

### TestAgent

| 属性 | 值 |
|------|-----|
| 名称 | TestAgent |
| 端口 | 8091 |
| 工作目录 | /root/.copaw_test (容器内) |
| 容器 | copaw-test |
| 角色 | 测试执行、质量保障 |

**职责**：
- 执行功能测试
- 执行性能测试
- 执行压力测试
- 生成测试报告
- 通知主 Agent 测试结果

## 开发工作流

```
1. 需求阶段
   Derek → Co (提出需求)

2. 开发阶段
   Co → DevAgent (分配开发任务)
   DevAgent 开发实现 → push 到 Fork 仓库

3. 测试阶段
   DevAgent → Co (开发完成)
   Co → TestAgent (执行测试)
   TestAgent → 测试目标容器 (运行测试)

4. 报告阶段
   TestAgent → Co (测试报告)

5. 验收阶段
   Co → Derek (请求验收)
   Derek 决策

6. 部署阶段
   Co 升级生产环境
```

## 关键设计原则

1. **完全独立**：每个 Agent 独立工作目录、SOUL.md、sessions
2. **通过 Co 协调**：DevAgent 和 TestAgent 不直接通信
3. **Docker 隔离**：DevAgent 和 TestAgent 在容器内
4. **Git 仓库持久化**：放在宿主机，通过挂载访问
