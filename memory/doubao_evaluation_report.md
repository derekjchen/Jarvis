## 四、API 与部署

### 4.1 接入方式
- **火山方舟平台**: https://console.volcengine.com/ark
- **API 兼容**: OpenAI 格式兼容
- **SDK**: Python/Go/Java

### 4.2 部署选项
1. **在线推理**: 按token计费，弹性扩缩
2. **低延迟在线推理**: 更快响应，价格稍高
3. **批量推理**: 价格低至在线的50%
4. **专属模型单元**: 预付费，更高并发

### 4.3 精调能力
- 支持 LoRA 精调: 40元/百万token
- 支持全量精调: 80元/百万token
- 基础模型: doubao-seed-1.6, doubao-1.5-pro-32k 等

## 五、与 Co (当前Agent) 的适配性分析

### 5.1 Agent 核心需求
1. **工具调用**: ✅ 支持
2. **长上下文**: ✅ 支持 256k
3. **多轮对话**: ✅ 支持
4. **推理能力**: ✅ thinking 系列支持深度推理
5. **多模态**: ✅ vision 系列支持

### 5.2 推荐模型配置

**场景一：日常对话 + 工具调用**
- 推荐: `doubao-seed-2.0-lite` 或 `doubao-seed-1.6`
- 理由: 平衡性能和成本，256k上下文足够

**场景二：复杂推理任务**
- 推荐: `doubao-1.5-thinking-pro` 或 `doubao-seed-1.6-thinking`
- 理由: 深度思考能力，适合复杂问题

**场景三：大量低成本调用**
- 推荐: `doubao-seed-2.0-mini` 或 `doubao-seed-1.6-flash`
- 理由: 极低价格，适合高频场景

**场景四：代码生成**
- 推荐: `doubao-seed-2.0-code`
- 理由: 代码专精

### 5.3 潜在问题
1. **无法直接测试**: 豆包官网需要登录，无法通过Agent直接对话测试
2. **性能数据缺失**: 官方文档未提供详细基准测试分数
3. **第三方评测少**: 需要实际使用来验证
