### M4.0 LLM 提取器修复 (2026-03-21 07:28-07:39)

**问题**: GLM-5 模型返回流式响应，每个 chunk 包含增量文本

**错误日志**:
```
LLM extraction failed: '\n    "type"'  # JSON 模板中 {} 被误解析
Failed to parse LLM response: Invalid control character
```

**根本原因**:
1. Prompt 模板中的 JSON 示例包含 `{` 和 `}` 字符，被 Python 误解析
2. 流式响应的每个 chunk 是增量文本，拼接后产生无效 JSON
3. GLM-5 返回 `content` 是列表，包含 `thinking` 和 `text` 类型

**修复方案**:
1. 模板中的 `{` 和 `}` 转义为 `{{` 和 `}}`
2. 只提取最后一个 chunk 的完整 `text` 内容
3. 过滤 `thinking` 类型，只取 `text` 类型的内容
4. 添加控制字符清理和正则 JSON 提取

**验证结果**:
```
消息: 我正在做一个卫星计算项目
提取成功: [{"type": "project", "name": "卫星计算项目", "content": "我正在做一个卫星计算项目"}]
```
